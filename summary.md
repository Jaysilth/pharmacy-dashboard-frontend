# Pharmacy Dashboard Frontend Summary

## Overview
The **pharmacy‑dashboard** project is a modern, TypeScript‑based React frontend for managing pharmacy operations. It is built with Vite, Tailwind CSS, and a collection of UI primitives from Radix UI and custom components. The application is responsive, supports light/dark mode, and integrates with a backend via React Query.

## Tech Stack
- **Framework**: React 19 (via `react` and `react‑dom`)
- **Language**: TypeScript (ESM module)
- **Build tool**: Vite 8
- **Styling**: Tailwind CSS 4 + `tw-animate-css`
- **State / Data fetching**: `@tanstack/react-query`
- **Form handling**: `react-hook-form` with Zod validation (`@hookform/resolvers`)
- **UI components**: Radix UI (AlertDialog, Dialog, Label, Select, Tooltip, Toast, etc.) and custom primitives under `src/components/ui`
- **Routing**: `wouter`
- **Utilities**: `date-fns`, `lucide-react`, `class-variance-authority`, `clsx`

### Key Directories & Files
- **`src/main.tsx`** – Mounts the React app, sets up `BrowserRouter` (`wouter`) and `ReactQueryClientProvider`.
- **`src/components/ui/`** – Core UI primitives such as `button.tsx`, `toast.tsx`, `dialog.tsx`, `tooltip.tsx`, and related Tailwind‑styled components. These are used throughout the app to keep a consistent look and feel.
- **`src/hooks/`** – Helper hooks:
  - `use-toast.ts` – Wrapper around Radix Toast for easy notifications.
  - `use-mobile.tsx` – Detects mobile viewport to enable responsive behavior.
- **`src/lib/utils.ts`** – General utility functions (date formatting, API helpers, etc.).
- **`src/pages/not-found.tsx`** – 404 fallback page.
- **`public/`** – Contains the favicon, Open Graph image, and robots.txt (served as‑is).
- **`dist/`** – Output directory after running `npm run build`.

## Development Tips
- Run `npm install` to install dependencies.
- Use `npm run dev` for hot‑reloading during development.
- The project follows a **component‑first** approach: most UI pieces live under `src/components/ui` and are composable.
- Tailwind classes are heavily used; refer to `tailwind.config.js` (generated by `@tailwindcss/vite`) for customizations.
- Form validation is handled via Zod schemas integrated with `react-hook-form`.
- Data fetching should go through a React Query client; add new queries in dedicated hooks or services.

## Build & Deploy
1. **Type‑check** – `npm run typecheck` ensures the TypeScript code compiles.
2. **Production build** – `npm run build` creates an optimized bundle in `dist/`.
3. **Preview** – `npm run preview` serves the static files; useful for sanity‑checking the production build locally before deployment.
